### 3. Design Patterns Used

- **Strategy Pattern**: Different payment processors (`PaymentProcessor` abstract class with concrete implementations)
- **Factory Pattern**: Order ID generation creates unique identifiers
- **Observer Pattern**: Notification service sends emails based on order state changes
- **Repository Pattern**: InventoryManager manages product data and stock
- **Decorator Pattern**: Discounts modify order totals without changing core logic
- **Singleton-like**: InventoryManager maintains state across the system
- **Command Pattern**: Order operations (create, cancel, pay) encapsulated as methods

The system follows good OOP principles with clear separation of concerns and modular design.
